# 数学直觉实验室章节修复复用规范

本文档用于后续逐章修复【数学直觉实验室】中 Taylor 之后的数学板块。它沉淀 Taylor 章节的修复模式，但不预先替每一章决定具体实验设计；每章开始修复时，按本模板填写并实施。

## 目标与适用范围

适用范围：

- `math-lab` 第 6-19 章中尚未精修的章节，例如 Monte Carlo、线性代数、LU、稀疏矩阵、条件数、特征值、Markov chains、有限差分、非线性方程、优化、最小二乘、SVD、PCA。
- 每章的正文、公式、双语内容、交互实验、Manim/Three/SVG/Canvas 可视化、quiz、误区卡片和视觉验收。

核心目标：

- 面向读者呈现一篇完整教程，不在页面上显式区分“原讲义”“GPT 精讲”“Cleaned Source”等来源。
- 保留原讲义的知识覆盖，同时把补充解释、直觉总结、例题和 ML 连接自然融入章节叙事。
- 每章必须具备完整 `zh-CN` / `en` 双语体验，不依赖第三方翻译服务或外部翻译 API。
- 公式、图、实验台和测验都应服务学习路径，而不是作为孤立补充块堆在末尾。

不做的事：

- 不为含公式或曲线的学习材料使用 image gen，避免公式文字和坐标失真。
- 不重新开启导入脚本中的第三方翻译路径。
- 不把内部工程来源说明暴露给读者页面。

## 章节修复模板

### 1. 章节定位

修复每章前先写清楚：

- 章节核心问题：读者学完本章要解决什么数学直觉问题？
- 先修关系：依赖 Taylor、向量矩阵、概率、优化中的哪些概念？
- ML 连接：它在机器学习、数值计算或模型训练中对应什么场景？
- 最小交付：至少包含一个核心公式、一个直觉解释、一个例题、一个互动实验或可视化读数、一个 quiz、一个误区卡片。

推荐格式：

```md
章节：<module id / title>
核心问题：<一句话>
读者应形成的直觉：<2-4 条>
ML / AI 连接：<1-3 条>
需要保留的原知识点：<列表>
需要补充的解释：<列表>
```

### 2. 内容融合结构

每章正文应重写为读者可顺读的章节结构，而不是按来源切块。标题应描述学习内容，例如：

- “从随机抽样到期望估计”
- “矩阵分解为什么能解线性方程”
- “条件数如何放大输入误差”
- “从协方差到主成分方向”

避免读者可见标题或标签：

- `原讲义整理`
- `GPT 精讲`
- `GPT-5.5 Enrichment`
- `Cleaned Source Note`
- `Complete note`
- `Source migration`

推荐章节顺序：

1. 动机与核心问题
2. 数学对象和公式
3. 几何/概率/数值直觉
4. 例题与误差或边界条件
5. ML / 数值库连接
6. 复习问题

如果某章内容较长，可以拆成 4-7 个二级章节；不要为了保留源结构而保留异常 heading。

### 3. 双语 i18n 要求

每章必须补齐以下字段的 `zh-CN` / `en`：

- 章节标题、subtitle、学习目标、AI/ML connection
- 概念卡片：名称、公式说明、变量说明、直觉、数值例子、模型连接
- 正文 sections
- visual title、transcript、learningPurpose、alt/caption
- lab title、控件文案、读数标签、状态解释、successCriteria
- quiz prompt、choices、explanation
- misconception statement、correction、example

中文模式要求：

- 不残留整段英文教学段落。
- 不使用明显机翻残留，例如“泰勒系列扩展包”“当时”这类语义错误片段。
- 必要英文术语可以保留，例如 Newton、Hessian、SVD、PCA、Monte Carlo，但要有中文解释。

英文模式要求：

- 不残留中文教学段落。
- 标题、按钮、读数和 quiz 均使用自然英文，不只翻译正文。

### 4. 公式与 Markdown 清理要求

统一通过 Markdown + KaTeX 渲染公式。正文中允许使用：

- 行内公式：`\( ... \)` 或 `$...$`
- 块级公式：`$$ ... $$` 或 `\[ ... \]`
- 表格、代码块、`details`、图片、引用块

清理重点：

- 修复 `## #`、重复 heading、空 heading、错级 heading。
- 修复单美元包裹的多行 display math。
- 清理 `<div>\[...\]</div>`、多余 `<br>`、无意义 HTML 包裹。
- 修复被导入脚本破坏的 LaTeX，例如错写的下标、矩阵、转义符。
- 渲染后页面正文不应出现裸 `\[`、`\]`、`\(`、`\)`、`$$`。
- 数学块在移动端必须可横向滚动，不能撑破页面。

### 5. 视觉/实验台规划

视觉策略优先级：

1. 确定性 SVG / Canvas / Vue 代码生成。
2. Manim 动画，用于逐步展示推导、曲线、矩阵变换、迭代轨迹。
3. Three.js，用于真正需要 3D 空间理解的概念。
4. image gen 只用于无文字、无公式、无坐标轴要求的装饰或封面图；数学材料默认不用。

插入位置：

- visual/lab 应穿插在相关章节之后，不默认堆到页面底部。
- 使用 `MathLabSection.visualIds` 和 `MathLabSection.labIds` 把资产放进正文流。
- 如果某章只有一个实验台，应放在核心公式或误差解释之后，让读者立即操作。

每个实验台至少包含：

- 2-4 个核心控件，直接映射公式变量或算法参数。
- 原始对象与当前近似/变换/估计的可视化。
- 3-5 个读数，例如误差、残差、估计值、迭代次数、条件数、解释方差。
- 中英文标签和状态解释。
- 移动端固定布局约束，避免控件和读数挤压重排。

### 6. ML 连接与例题要求

每章至少补充一个 ML / AI 连接，不要只写“机器学习会用到”。连接要具体：

- Monte Carlo：采样估计、dropout、随机优化、Bayesian 推断。
- 矩阵与范数：embedding 距离、权重矩阵、梯度范数。
- LU / 稀疏矩阵：线性系统、图结构、推荐系统、科学计算。
- 条件数：训练稳定性、特征缩放、病态最小二乘。
- 特征值/Markov chains：PageRank、稳定分布、谱方法。
- 有限差分/非线性方程/优化：梯度检查、Newton 法、求根、局部模型。
- 最小二乘/SVD/PCA：回归、降维、压缩、推荐、去噪。

例题要求：

- 至少一个手算例题或小型数值例子。
- 例题要包含解释，不只给公式。
- 如果涉及误差、收敛、稳定性，必须明确条件或失败场景。

### 7. Quiz 与误区卡片要求

每章至少包含：

- 2 个 quiz，优先使用 single-choice 或 numeric。
- 2 个 misconception cards。
- quiz explanation 和 misconception example 都必须支持公式渲染。

误区应针对真实学习风险，例如：

- “阶数越高一定越好”
- “矩阵可逆就一定数值稳定”
- “PCA 自动找到最能分类的方向”
- “Monte Carlo 样本翻倍误差就减半”

## 实现检查清单

### 数据层 override

- 为精修章节新增专用 override 构建函数，接收已导入模块并返回完整 `MathLabModule`。
- 保留 `id`、`order`、`sourceNoteFile`、`originalSort` 等内部追踪字段。
- 不改变路由结构，不破坏其他章节的导入数据。
- 正文 sections 使用融合后的章节结构，toc 从 sections 生成。

### 页面内联 visual/lab placement

- 需要在正文中插入视觉材料时，使用 `visualIds` / `labIds`。
- visual id 必须能在 `module.visuals` 中找到。
- lab id 必须能在 `module.labs` 中找到。
- 已内联的 visual/lab 不应再出现在底部 supplemental 区。

### 专用 lab 或通用 lab 选择

使用专用 lab 的条件：

- 该章核心概念需要特殊坐标系、曲线、矩阵、采样或迭代逻辑。
- 通用 `NumericalMiniLab` 无法表达关键误差、残差或几何直觉。

继续使用通用 lab 的条件：

- 该章只是需要一个简短数值读数或已有通用实验足够清楚。
- 暂时不做深度交互，但仍要保证双语和布局质量。

专用 lab 的计算逻辑应尽量抽到 `utils`，便于单元测试。

### Manim / Three / SVG 资产

- Manim scene 放入现有 Manim 管线，输出到 `public/manim/math-lab/`。
- 每个 mp4 配套 poster SVG 和 metadata。
- SVG/Canvas 曲线、矩阵、采样点必须确定性生成。
- 不提交 Manim 渲染缓存，例如 `media/`、`__pycache__/`。

### 样式与移动端

- 数学块和表格必须 `overflow-x: auto`。
- 固定格式组件要设置稳定尺寸或 `aspect-ratio`。
- 控件、图、读数在 390px 宽移动视口下不能横向溢出。
- 不要把页面段落包装成层层嵌套卡片；只有实验台、modal、重复 item 才使用 card 风格。

## 验收标准

### 单元测试

每章修复后至少补充或更新：

- 章节 `zh-CN` / `en` 标题、正文、目标、quiz、misconception 存在性。
- 中文正文不含明显机翻残留或整段英文。
- 英文正文不含中文教学段落。
- Markdown 全量渲染后不残留裸公式定界符，并且输出 KaTeX。
- 专用 lab 的核心计算函数有数值趋势测试。

### Build

运行：

```bash
npm test -- --test-reporter=spec
npm run build
```

若本轮只改文档，可不运行 build，但应至少检查 Markdown 文件存在且可读。

### 浏览器视觉检查

启动本地 Vite 后检查目标章节：

- 中文桌面视口。
- 英文桌面视口。
- 中文或英文移动视口，建议 390px 宽。
- 无横向滚动。
- 无公式裸定界符。
- visual / video / poster 可加载。
- 实验控件改变后读数或图形确实更新。
- 页面不出现面向内部的来源标签，例如 `原讲义`、`GPT 精讲`、`Cleaned Source`、`Complete note`。

### 中英文切换检查

至少验证：

- `document.documentElement.lang` 随语言切换变化。
- 中文模式不显示整段英文教学内容。
- 英文模式不显示中文教学标题或教学段落。
- quiz、误区卡片、实验台控件和读数同步切换语言。

- 提供完整 `zh-CN` / `en`：正文、学习目标、概念解释、visual 文案、lab 文案、quiz、误区卡片。

- 单元测试覆盖双语内容、公式渲染、核心计算逻辑。
