# ML 教学内容导入与补充规范

本文档用于后续持续补充【损失函数】【梯度下降】【线性回归】【逻辑回归】【MLP】五个机器学习教学板块。目标不是机械搬运公开资料，而是把 D2L、Google Machine Learning Crash Course、CS357、mlcourse.ai 和 OpenStax 的教学结构改写成本站统一的“数学直觉 + 手算例子 + 公式解释 + 常见误解 + 交互实验”格式。

## 1. 总原则

- 不按原资料顺序机械搬运，而是按本站学习路径重组。
- 每章必须回答一个明确学习问题，避免一章同时承担多个核心概念。
- 每章必须包含：核心问题、概念直觉、手算例子、公式、常见误解、交互实验设计、来源参考。
- 优先复用本站已有交互组件和仿真数据，再决定是否新增组件。
- 中文解释优先保证教学自然，英文解释保留准确术语，不逐句硬译。
- 所有章节保留来源映射，方便后续维护、二次改写和追溯。

## 2. 来源优先级

1. **D2L**：结构化、适合算法实现、训练流程、线性回归、优化、MLP、过拟合与正则化。
2. **Google Machine Learning Crash Course**：适合初学者直觉、loss、log loss、分类概率、训练诊断和泛化。
3. **CS357**：适合补线性代数、最小二乘、优化数学基础、数值稳定性。
4. **mlcourse.ai**：适合补逻辑回归、最大似然、线性分类、正则化推导。
5. **OpenStax Principles of Data Science**：适合补神经网络、反向传播和浅层直觉解释。

## 3. 每章改写模板

```md
## 章节标题

### 核心问题
这一章要回答学生脑中的哪一个问题。

### 概念直觉
用图像化、任务化语言解释，不直接堆公式。

### 手算例子
给一个小数据例子，让学生能手工算出关键数值。

### 公式
只放本章必要公式，并解释每个符号的含义。

### 常见误解
指出最容易混淆的概念。

### 交互实验设计
说明页面上应如何拖动、切换、观察。

### 来源参考
列出参考材料章节或页面。
```

## 4. 五大板块导入清单

| 板块 | 建议章节 | 主参考 | 实验目标 |
|---|---|---|---|
| 损失函数 | 为什么需要 loss；MSE/MAE；BCE/交叉熵；似然；负对数；MLE 桥接 | Google MLCC + D2L | 观察 error 如何被不同规则改写成 loss，理解 NLL/MLE 来源 |
| 梯度下降 | loss surface；等高线；负梯度；学习率；鞍点/局部最优；batch 噪声 | D2L + Google MLCC | 观察轨迹、震荡、发散、噪声和收敛状态 |
| 线性回归 | 直线含义；残差/MSE；梯度训练；模型边界；多元回归；多项式；过拟合；正则化 | D2L + CS357 | 连接数据空间中的线/面与参数空间中的优化 |
| 逻辑回归 | 线性打分；sigmoid 概率；交叉熵；正则化；线性不可分限制 | Google MLCC + mlcourse.ai | 观察边界移动、概率背景、log loss 和 XOR 失败 |
| MLP | 隐藏层特征；激活函数；输出层；反向传播；容量；泛化 | D2L + OpenStax | 观察隐藏空间重组、非线性边界和容量/过拟合权衡 |

## 5. 板块衔接规则

- 【损失函数】是课程入口，重点回答“模型好不好怎样变成一个数”。
- 【梯度下降】接在损失之后，重点回答“有了目标函数后参数怎样移动”。
- 【线性回归】把 loss 和优化落到连续值预测，重点回答“斜率、截距和 MSE 怎样一起工作”。
- 【逻辑回归】把线性模型转向分类，重点回答“线性打分怎样变成概率和边界”。
- 【MLP】从逻辑回归的线性限制出发，重点回答“隐藏层怎样重组空间并扩展表达能力”。

## 7. 验收标准

- 学生能说出本章解决的核心问题。
- 学生能手算一个最小例子。
- 学生能解释公式中每个主要符号的含义。
- 学生能通过实验观察到公式对应的行为。
- 学生能避开至少一个常见误解。
- 每章能追溯到至少一个来源材料或站内前置章节。
- 五个板块顺序保持为：损失函数 -> 梯度下降 -> 线性回归 -> 逻辑回归 -> MLP。
