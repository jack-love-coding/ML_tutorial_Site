# 第 3 课：矩阵与批量计算——两条关卡记录怎样一起预测

> **建议学习时间：60–90 分钟**
>
> **本课主线：**把 `x = [2, 3]` 与第二个样本 `[1, 4]` 按行堆成 `X`，用同一 `w = [4, -1]`、`b = 5` 计算两条关卡完成时长。
>
> **学习方式：**每一次矩阵运算先写 shape，再逐行展开；练习不计分。

## 1. 开场：为何不是写两遍点积？ {#matrices-opening}

两个样本当然可以逐个调用函数，但一百个、一百万个呢？批量计算的价值不只是少写循环，更重要的是明确“行是样本、列是特征”的数据合同，并让同一参数作用于每一行。

本课的问题是：为何 `(2,2) @ (2,)` 得 `(2,)`？偏置怎样加到两个预测而不是四个单元格？若矩阵转置，数学仍能算时，语义是否仍正确？

## 2. 回顾：向量点积与行列读取 {#matrices-recap}

上一课的单样本规则是 $\hat y=\mathbf w^T\mathbf x+b$。$[2,3]\cdot[4,-1]=5$，加偏置得 10。矩阵只是把多个同结构向量按行排列。

读二维表时先说位置：第 $i$ 行第 $j$ 列 $X_{ij}$，$i$ 选择样本，$j$ 选择特征。主线中特征列必须始终为“基础任务数、提示请求数”。

## 3. 主线连接：Xw+b 的 shape 账本 {#matrices-shared-task}

核心词汇继续使用 `x`, `w`, `b`, `y_hat`, `y`, `L`。批量时用大写 `X` 表示许多 `x`：

```text
X = [[2, 3], [1, 4]]   shape (2, 2)
w = [4, -1]            shape (2,)
b = 5                  scalar
y_hat = [10, 5]        shape (2,)
y = [9, 7]             shape (2,)
L = 2.5                scalar MSE
```

合同为 `y_hat = Xw + b`。其中内侧特征维 2 对齐，外侧样本维 2 被保留。目标不参与 `Xw+b`，只进入 MSE。

## 4. 直觉：表格、流水线与图像网格 {#matrices-intuition}

把 $X$ 想成表格：每行是一张关卡卡片，每列是同一种测量。$w$ 像对每列应用的读取规则。矩阵乘向量就是让每一行独立完成同一个点积。

矩阵也可表示图像网格变换。例如纵向翻转坐标的矩阵会保留横坐标、改变纵坐标符号。这是理解“线性变换”的辅助侧视图；主线 $X$ 是数据批，不是变换矩阵，二者不能因都写方括号而混为一谈。

## 5. 正式表达：行、列、乘法与广播 {#matrices-formal}

若 $X\in\mathbb R^{n\times d}$、$w\in\mathbb R^d$，则 $Xw\in\mathbb R^n$。第 $i$ 个输出为

$$
(Xw)_i=\sum_{j=1}^{d}X_{ij}w_j.
$$

$n$ 是样本数，$d$ 是特征数。本例 $n=d=2$ 恰好相等，但职责不同；换成 3 个样本时 $X$ 为 `(3,2)`，$w$ 仍为 `(2,)`。

标量 $b$ 加到每个预测：$\hat y_i=(Xw)_i+b$。这是标量广播，不是把 $b$ 加进 $X$ 每个特征后再点积；两者一般不同。

MSE 为

$$
L=\frac1n\sum_{i=1}^{n}(\hat y_i-y_i)^2.
$$

### 深入一：内维、偏置与损失轴

一般 shape 是 `(n,d) @ (d,) -> (n,)`。两个 $d$ 必须相等，因为每行的每个特征都要有权重；$n$ 被保留，因为每个样本都要有输出。本例 $n=d=2$ 只是巧合。新增第三个样本后 `(3,2) @ (2,) -> (3,)`；新增第三个特征则必须让 $X$ 与 $w$ 同时扩展特征维。

偏置在点积后广播。若误写 `(X+b)@w`，第一行变为 `[7,8]@[4,-1]=20`。一般地，$(x+b\mathbf1)^Tw=x^Tw+b\sum_jw_j$，并不等于 $x^Tw+b$。广播能够执行，不代表运算顺序符合模型。

损失也有轴：残差 `[1,-2]` 逐元素平方为 `[1,4]`，再沿样本轴平均为 2.5。保留逐样本平方误差可以看出第二行贡献 4；只有总体数值会丢失诊断位置。

### 深入二：行列语义的审计表

| 样本 | 第 1 列贡献 | 第 2 列贡献 | 点积 | 加偏置 | 目标 | 平方误差 |
|---|---:|---:|---:|---:|---:|---:|
| 1 | $2\times4=8$ | $3\times(-1)=-3$ | 5 | 10 | 9 | 1 |
| 2 | $1\times4=4$ | $4\times(-1)=-4$ | 0 | 5 | 7 | 4 |

从左到右每栏只做一个动作。第一行若不复现上一课，检查数据迁移与列序；点积正确但预测错误，检查偏置；预测正确而损失错误，检查目标的行对齐与平方平均。

总量还有辅助核验：贡献矩阵所有元素和为 $8-3+4-4=5$，两个点积和也是 5。它不能替代逐行结果，却能捕捉漏项。数据集应同时保存样本 ID；交换 $X$ 行时，$y$ 与 ID 必须做同一置换。

## 6. 算例一：逐 shape 展开整批预测 {#matrices-worked-shared}

先锁定 `X: (2, 2)`、`w: (2,)`、`Xw: (2,)`。逐行：

$$
Xw=
\begin{bmatrix}2&3\\1&4\end{bmatrix}
\begin{bmatrix}4\\-1\end{bmatrix}
=\begin{bmatrix}2\times4+3\times(-1)\\1\times4+4\times(-1)\end{bmatrix}
=\begin{bmatrix}5\\0\end{bmatrix}.
$$

广播偏置：`y_hat = [10, 5]`。与 `y=[9,7]` 比较，残差 `[1,-2]`，平方误差 `[1,4]`，所以 `L = 2.5`。

第一行完整复现上一课；第二行预测比目标低 2 分钟。平均不是把残差先平均再平方，否则 $(1-2)^2/4=0.25$，会错误地让正负抵消。

## 7. 算例二：网格点的线性镜像 {#matrices-worked-auxiliary}

辅助变换取 `[[1, 0], [0, -1]]`，它把二维列向量 $(u,v)$ 映射到 $(u,-v)$。对点 `[2, 3]`：

$$
\begin{bmatrix}1&0\\0&-1\end{bmatrix}
\begin{bmatrix}2\\3\end{bmatrix}
=\begin{bmatrix}2\\-3\end{bmatrix}.
$$

纯文本证据是 `[2, 3] -> [2, -3]`。若把四个角点 `(1,1),(-1,1),(-1,-1),(1,-1)` 都变换，会得到关于横轴翻转后的网格。原点保持不变、直线仍映成直线。

注意这里矩阵位于乘号左侧并扮演变换；主线中 $X$ 的每一行是样本。判断角色要看公式与 shape，而不是看变量是否是二维数组。

## 8. 代码翻译：向量化但仍可逐行核验 {#matrices-code}

```python
import numpy as np

X = np.array([[2.0, 3.0], [1.0, 4.0]])
w = np.array([4.0, -1.0])
b = 5.0
y = np.array([9.0, 7.0])

assert X.ndim == 2 and w.ndim == 1
assert X.shape[1] == w.shape[0]
weighted = X @ w
y_hat = weighted + b
residuals = y_hat - y
L = np.mean(residuals ** 2)
print(weighted, y_hat, L)  # [5. 0.] [10. 5.] 2.5
```

调试时可用 `X * w` 得到每行每列贡献，再沿 `axis=1` 求和。若误用 `axis=0`，会按列汇总并失去“每样本一个预测”的合同。

### 代码基线与 shape 症状

```python
loop_predictions = []
for row_index, x in enumerate(X):
    assert x.shape == w.shape
    weighted_sum = 0.0
    for feature, weight in zip(x, w):
        weighted_sum += feature * weight
    loop_predictions.append(weighted_sum + b)
assert np.allclose(loop_predictions, y_hat)
```

外循环跨样本，内循环跨特征。`X @ w` 没有改变这个数学层次，只把循环交给数组实现。对固定小数据保留相等断言，可以发现 axis 或广播错误。

| 症状 | 常见原因 | 最小检查 |
|---|---|---|
| 内维报错 | 特征数与权重数不同 | `X.shape[1] == w.shape[0]` |
| 残差变 `(2,2)` | `(2,)` 与 `(2,1)` 交叉广播 | `y_hat.shape == y.shape` |
| 数值顺序交换 | 样本行交换 | 样本 ID 与目标同步置换 |
| shape 正常但首项非 10 | 列顺序错位 | 展开第一行贡献 |

最后一种最危险，因为程序能够运行。shape 是必要证据，列语义是另一份必要证据。

## 9. 控制实验：只增加或交换一行 {#matrices-experiment}

先固定 $w,b$，只新增第三行 `[3,2]`。新矩阵 shape `(3,2)`，新行点积 $12-2=10$，预测 15；原两项预测仍是 10、5。新增样本不应改变旧样本的前向结果。

再做独立实验：只交换前两行。输出也应交换为 `[5,10]`。这称为行置换一致性：行顺序改变样本排列，不改变各样本内部计算。

若只交换两列却不交换 $w$，第一行变为 $3\times4+2\times(-1)=10$，预测 15。数值能算但语义错位，是比 shape 报错更危险的问题。

### 实验扩展：新增样本怎样改变汇总指标

若第三行 `[3,2]` 的目标是 14，则预测 15、平方误差 1，三样本 MSE 为 $(1+4+1)/3=2$。旧两行预测没有变化，但总体 $L$ 从 2.5 到 2，因为平均集合和分母改变。要区分“模型对旧样本的行为”与“汇总指标的成员”。

若第三行目标是 5，平方误差为 100，MSE 变成 35。高误差样本会显著影响均方误差，这提示检查目标错位、测量异常或任务设定；不能为了让指标好看就删除样本。本课只观察，不引入新损失。

交换行实验中，若只交换 $X$ 而不交换 $y$，前向预测只是换序，但比较阶段把不同关卡配在一起。正确流程是用同一索引排列特征、目标与样本 ID。新增一列时则必须同时更新列名与 $w$，这与新增一行的规则不同。

做完实验请用三句话报告：改了哪一轴；哪些单样本结果应保持；为何总体 $L$ 可能改变。若只能说“矩阵变大了”，说明 shape 尚未连接到样本/特征语义。

## 10. 误区诊断：能运行不代表批量合同正确 {#matrices-misconceptions}

### 进入误区前：完整走查一个三样本批次

把第三行 `[3,2]` 与目标 14 加入后，先写：`X (3,2)`、`w (2,)`、`Xw (3,)`、`b scalar`、`y_hat (3,)`、`y (3,)`、`L scalar`。内维 2 对齐，样本维 3 保留。

贡献矩阵为

$$
\begin{bmatrix}
8&-3\\4&-4\\12&-2
\end{bmatrix},
$$

沿特征轴求和得到 `[5,0,10]`，加偏置得到 `[10,5,15]`。目标 `[9,7,14]` 给残差 `[1,-2,1]`、平方 `[1,4,1]`、MSE 2。这条走查刻意同时保留 shape 与值，防止“只会推 shape”或“只会算数字”。

若采用显式循环，外循环运行三次，每次内循环运行两次；若采用矩阵乘法，数学上仍有六次配对乘法。向量化改变执行表达，不改变样本和特征的对应关系。性能优势来自底层连续数组与优化实现，不来自减少必要的数学贡献。

### 图像网格辅助例的边界

可以把四个角点按行堆成矩阵 $P$，再统一做坐标变换。若点采用行向量，常写 $P A^T$；若点采用列向量，常写 $AP$。两种约定都可使用，但不可在同一推导中无声切换。主线采用“样本为行”，所以 `X @ w` 保留行；辅助图形也应先声明点的摆放方式。

镜像矩阵的第一列说明基向量 `[1,0]` 映到哪里，第二列说明 `[0,1]` 映到哪里。当前矩阵让第一基向量不变、第二基向量取负，因此任意点 $(u,v)=u(1,0)+v(0,1)$ 映到 $(u,-v)$。这把逐点算例推广为可解释规则，而不是只记一个结果。

然而图像变换的矩阵 $A$ 与数据矩阵 $X$ 角色不同：$A$ 的列编码变换对基方向的作用，$X$ 的行编码关卡样本。若把“二维数组”都叫变换，会遮蔽批量数据的行语义。判断角色要同时看变量来源、公式位置和输出 shape。

### 批量调试的二分策略

当一千行预测中只有部分异常，先比较循环基线与向量化结果，找出第一个不一致行；再只打印该行的列名、特征、权重与贡献。若所有行都按同一比例错，优先检查偏置或单位缩放；若只有某列相关样本错，检查列清洗与顺序；若预测正确但损失错，检查目标对齐。

调试日志不要倾倒完整大矩阵。记录 shape、有限值计数、前两行、异常行索引和逐列贡献，既足够审计，也减少认知负担。本课小数据允许全部打印，是为了建立以后缩小问题范围的方法。

### 阅读检查点

现在遮住正文，回答：为什么新增一行不要求改变 $w$？为什么新增一列必须改变 $w$？为什么交换行要同步交换目标，而交换列要同步交换权重？三个答案分别对应样本独立、特征配对与监督对齐。若其中一个说不清，回到 shape 账本并给每条轴写名字。

再检查 MSE：它把样本维汇总成标量，因此不能从 2.5 单独恢复 `[1,4]`。需要诊断具体样本时，必须保留未汇总损失。这也是“聚合越早，信息丢失越早”的一般程序设计原则。

### 误区一：方阵的两个 2 含义相同

**为什么容易发生：**本例 shape 恰好 `(2,2)`。

**本例诊断：**一个 2 是样本数，一个 2 是特征数；增加样本只改变前者。

**修复动作：**总写 `(n_samples, n_features)`，不用裸 `(2,2)` 结束解释。

### 误区二：转置后仍可算就仍正确

**为什么容易发生：**方阵转置不改变 shape。

**本例诊断：**$X^T$ 的行变成特征列集合，`X.T @ w` 不再逐样本预测。

**修复动作：**在 shape 外检查行列语义，并逐行核验第一项。

### 误区三：先平均残差再平方就是 MSE

**为什么容易发生：**“平均平方”与“平方平均”词序相近。

**本例诊断：**正确结果 2.5；错误方法 0.25，正负残差被抵消。

**修复动作：**保留 `residuals -> squared -> mean` 三个中间数组。

## 12. 小结与交接：把批量误差交给导数课 {#matrices-handoff}

本课把一个样本点积扩展为逐行批量计算，并用每一步 shape 保证语义。导数课直接接收 `predictions = [10, 5]`、`targets = [9, 7]`、残差 `[1,-2]` 与 MSE 2.5。

建议学习节奏是：15 分钟写清行与列的协议，20 分钟展开主线和网格两套算例，20 分钟比较循环与向量化，最后 20–30 分钟完成控制实验、误区和练习。若某个 NumPy 表达式无法逐行展开，就先回到审计表；若数值正确却说不清哪一轴是样本，就先回到 `(n,d)` 记号。两类理解缺一不可。

进入下一课前，用第三行 `[3,2]` 做压力测试：预测应追加 15，原两项不变；目标 14 时 MSE 变 2；交换行时预测、目标和 ID 必须同步交换。它同时检查特征内维、样本外维与监督对齐。只有这三项稳定，改变参数后观察损失才有可靠的数据基础。

还可以把主线表复制到纸上，用两种颜色分别圈出“沿行汇总特征”和“沿样本汇总损失”。若不能使用颜色，就用实线框标特征汇总、虚线框标样本汇总，并写文字标签。关键结论不能依赖颜色：矩阵乘法先为每行消去特征轴，MSE 后来才消去样本轴。两次汇总发生在不同阶段、回答不同问题。

最后核对单位：每个 $X_{ij}w_j$ 都必须是分钟，同行贡献才能相加；偏置也是分钟；残差是分钟；平方误差和 MSE 是分钟²。shape 正确但单位不可加，同样说明模型合同有误。

将 shape、单位和数值三列同时核对，才能把“程序可运行”提升为“计算可解释、可复现”。
