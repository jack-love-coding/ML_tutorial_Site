# 第 2 课：向量与样本表示——一次预测如何读懂多个特征

> **建议学习时间：60–90 分钟**
>
> **本课主线：**继续预测同一个小游戏关卡的预计完成时长。`x = [2, 3]`、`w = [4, -1]`、`b = 5`，预测 `y_hat = 10` 分钟，目标 `y = 9` 分钟。
>
> **学习方式：**每遇到紧凑符号都先展开为配对乘法；练习不计分，用于发现维度、顺序与单位上的理解缺口。

## 1. 开场：两个数为何能代表一个样本？ {#vectors-opening}

上一课把 `features = [2, 3]` 当作列表：2 个基础任务、3 次提示请求。现在请思考：为什么它可以写成一个整体 $\mathbf x$？若交换成 `[3, 2]`，还是同一个样本吗？为什么 `[4, -1] * [2, 3]` 不能直接叫预测？

真正的问题不是“会不会画箭头”，而是能否维护一份表示合同：每个位置有固定语义，维度一致，计算逐项配对，最终得到一个可解释输出。本课会先用带单位的主线解释贡献，再用独立、无量纲的几何例子学习长度与投影。

先写猜想：$\mathbf w$ 更像“另一个样本”，还是“把各特征换算为分钟贡献的规则”？当 $\mathbf w^T\mathbf x=5$ 时，这个 5 的单位是什么？保留答案到实验节。

## 2. 回顾：有序列表、坐标与平方误差 {#vectors-recap}

向量课只需要三件旧知识。第一，列表有顺序：`x[0]` 始终对应基础任务数，`x[1]` 始终对应提示请求数。第二，坐标 $(2,3)$ 可画成从原点指向点 $(2,3)$ 的箭头；箭头只是表示法，不会改变数据。第三，本单元采用

$$
L=(\hat y-y)^2.
$$

平方使预测偏高与偏低都产生非负误差。主线样本中 $\hat y=10,y=9$，所以 $L=1$。这里先计算，不讨论怎样自动改变参数。

还要区分标量与向量：5、9、10 是标量；$[2,3]$ 和 $[4,-1]$ 是二维向量。标量只有一个位置，向量则是一组按协议排列的数。

## 3. 主线连接：向量把同一关卡的特征绑在一起 {#vectors-shared-task}

核心词汇始终是 `x`, `w`, `b`, `y_hat`, `y`, `L`：

```text
x = [2, 3]       当前关卡样本，shape (2,)
w = [4, -1]      贡献系数，shape (2,)
b = 5            偏置，标量
y_hat = 10       预计完成时长，标量，单位分钟
y = 9            实际完成时长，标量，单位分钟
L = 1            当前样本平方误差，标量，单位分钟²
```

紧凑合同是 `y_hat = w^T x + b`。这里 $w$ 是作用在特征向量上的**带单位线性泛函**（也可读作贡献系数表），不是几何箭头。具体地，$w_1=4$ 分钟/基础任务，$w_2=-1$ 分钟/提示请求。上标 $T$ 只提醒逐坐标配对并汇总：

$$
\mathbf w^T\mathbf x=w_1x_1+w_2x_2.
$$

目标 $y$ 不进入点积。顺序固定为“先用 $x,w,b$ 得到 $y\_hat$，再用 $y\_hat,y$ 得到 $L$”。这条信息边界会贯穿后续课程。

## 4. 直觉：每个系数把一个特征换算成分钟 {#vectors-intuition}

把 $x=[2,3]$ 看成一张有两栏的关卡记录，而不是空间箭头。第一栏的 2 个基础任务由 $w_1=4$ 分钟/基础任务换算成 8 分钟；第二栏的 3 次提示由 $w_2=-1$ 分钟/提示请求换算成 -3 分钟。两项现在都以分钟计，才可以相加。

因此 $w$ 的主线作用是把 $x$ 映射到一个标量：$x\mapsto w^Tx$。这种“输入向量到标量、且保持加法与数乘”的规则叫线性泛函。初学阶段不必背术语，但要守住它的操作含义：只计算各项 $w_jx_j$，再计算 $w^Tx=\sum_jw_jx_j$。

若 $x_1$ 增加 1 而其余不变，贡献增加 4 分钟；若 $x_2$ 增加 1，贡献减少 1 分钟。这里的正负号描述贡献变化，不需要借助角度或投影。偏置 $b$ 不是某个特征的贡献，它在汇总之后单独加上。

## 5. 正式表达：维度、线性泛函与单位合同 {#vectors-formal}

一个 $d$ 维实向量写为 $x\in\mathbb R^d$。本例 $d=2$，维度就是有多少个按协议排列的特征：

$$
\dim(x)=2.
$$

点积要求维度相同：若 $\mathbf x,\mathbf w\in\mathbb R^d$，则

$$
\mathbf w^T\mathbf x=\sum_{j=1}^{d}w_jx_j\in\mathbb R.
$$

输入是两个长度为 $d$ 的向量，输出是标量。若 $\mathbf x=[2,3,1]$ 而 $\mathbf w=[4,-1]$，第三项没有权重，规则不完整。不能默默截断，也不能凭空补零。

形式上，固定 $w$ 后，$f_w(x)=w^Tx$ 满足 $f_w(x+z)=f_w(x)+f_w(z)$ 与 $f_w(cx)=cf_w(x)$，所以它是线性泛函。加上偏置后的 $w^Tx+b$ 是仿射映射。主线只使用这个代数合同，不计算欧氏几何量。

### 深入一：顺序、单位与三种点积读法

向量坐标顺序属于数据接口，而不是排版选择。若第一列约定为基础任务、第二列为提示请求，那么把 `[2,3]` 改成 `[3,2]` 却继续使用 `[4,-1]`，得到的是语义错位的新记录。若确实交换两列，权重也必须按同一置换交换：

$$[4,-1]^T[2,3]=[-1,4]^T[3,2]=5,$$

但 $[4,-1]^T[3,2]=10$。点积对共同置换不变，对单边置换不变这一说法是错的。调试时应打印列名、特征值、权重与贡献四列，而不是只打印两个裸数组。

尺度变换仍需单位账本。若把“基础任务数”从个改成十个一组，2 变成 0.2，对应系数应从 4 变成 40 分钟/组，乘积仍为 8 分钟。参数值必须连同特征单位解释。

同一计算有两种主线核验读法：配对贡献求和；行向量乘列向量 `(1,2) @ (2,1) -> (1,1)`。两者数值都应为 5，但返回容器可能是标量 5 或二维 `[[5]]`；实现必须规定一种接口。

### 深入二：代码 shape 与透明基线

`np.array([2,3])` 是 `(2,)`，`[[2,3]]` 是 `(1,2)`，`[[2],[3]]` 是 `(2,1)`。元素相同不代表运算合同相同。本课单样本统一采用 `(2,)`，所以点积直接返回标量。

```python
def dot_checked(w, x):
    if len(w) != len(x):
        raise ValueError("w and x must have the same dimension")
    total = 0.0
    for index, (weight, feature) in enumerate(zip(w, x)):
        contribution = weight * feature
        print(index, feature, weight, contribution)
        total += contribution
    return total
```

主线会打印 8、-3 并返回 5。这个慢但透明的版本是 NumPy `@` 的独立基线。输入还需检查有限值；`NaN` 或无穷值会穿过每个乘积，最后使预测与损失都不可解释。

## 6. 算例一：完整点积、预测与误差 {#vectors-worked-shared}

写下可核查纯文本：

```text
w^T x = 4*2 + (-1)*3 = 5
y_hat = 5 + 5 = 10
L = (10 - 9)^2 = 1
```

逐步看：$w_1x_1=4\times2=8$ 分钟，$w_2x_2=(-1)\times3=-3$ 分钟，净加权贡献为 $8-3=5$ 分钟；偏置 $b=5$ 分钟使预测为 10 分钟；目标 $y=9$ 分钟只在最后进入平方误差。

贡献表能定位符号错误：

| 坐标 | 特征 | 权重 | 乘积贡献 |
|---|---:|---:|---:|
| 1 | 2 | 4 | 8 |
| 2 | 3 | -1 | -3 |
| 汇总 | — | — | 5 |

核验输出形状：逐元素乘积 `[8, -3]` 是 `(2,)`；求和后 5 是标量；加偏置仍是标量。若代码停在 `[8, -3]`，计算尚未完成。

## 7. 算例二：无量纲同尺度位移的几何投影 {#vectors-worked-auxiliary}

现在离开关卡预测，建立一个独立几何场景。平面网格的横纵坐标都用同一尺度的**无量纲网格单位**。位移 `u = [3, 4]`，参考方向 `v = [4, 0]`。因为两个坐标同单位、同尺度，欧氏长度、夹角与投影才有直接几何意义。

长度为 `||u|| = 5`、$\lVert v\rVert=4$。点积 $u^Tv=12$，所以

$$
\operatorname{proj}_{v}(u)=\frac{u^Tv}{v^Tv}v
=\frac{12}{16}[4,0]=[3,0].
$$

纯文本核验为 `proj_v(u) = [3, 0]`。剩余位移 $u-[3,0]=[0,4]$ 与 $v$ 点积为 0，说明它垂直于参考方向。夹角余弦为 $12/(5\times4)=0.6$。

适用条件必须同时满足：坐标表示同一几何空间；各轴单位兼容；尺度已经统一。若横轴是米、纵轴是秒，或一个轴被任意放大十倍，就不能直接把这个投影视为物理长度。这个辅助例只教授欧氏几何，不替代主线的带单位贡献计算。

## 8. 代码翻译：保留 shape 与中间贡献 {#vectors-code}

```python
import numpy as np

x = np.array([2.0, 3.0])
w = np.array([4.0, -1.0])
b = 5.0
y = 9.0

assert x.shape == w.shape == (2,)
contributions = w * x
weighted_sum = w @ x
y_hat = weighted_sum + b
L = (y_hat - y) ** 2

print(contributions)  # [ 8. -3.]
print(weighted_sum)   # 5.0
print(y_hat, L)       # 10.0 1.0
```

`*` 做逐元素乘法，`@` 做点积。为了调试，可先保留两者；生产表达式可写 `y_hat = w @ x + b`。主线代码到此为止；几何辅助量只在独立 $u,v$ 例中出现。

若输入包含 `NaN` 或无穷值，结果会污染整条链。可先检查 `np.isfinite(x).all()`；若 shape 不同，明确报错，不用 `zip` 静默截断。

## 9. 控制实验：只改变贡献系数 {#vectors-experiment}

固定 $x=[2,3],b=5,y=9$，比较四个权重：

| $w$ | $w^Tx$ | $\hat y$ | $L$ | 观察 |
|---|---:|---:|---:|---|
| `[4,-1]` | 5 | 10 | 1 | 基线 |
| `[8,-2]` | 10 | 15 | 36 | 两项贡献系数都放大两倍 |
| `[-4,1]` | -5 | 0 | 81 | 两项贡献符号都反转 |
| `[3,-1]` | 3 | 8 | 1 | 只改第一权重 |

这次一次只选择一种操作。整体缩放会把两个 $w_jx_j$ 同比例缩放；取负会反转每项贡献符号；只改一项只改变对应特征贡献。预测变大不等于损失变小，评价必须回到目标 9。

建议画贡献条形图并同时记录表格：每根条标明特征、系数、乘积和分钟单位。不要从单个样本推断某个系数“全局最好”。

### 实验记录、变量隔离与形成性反馈

每轮实验应记录：改变因素、固定因素、点积、预测、目标、损失。以整体放大权重为例，应写“只把 $w$ 乘 2；$x,b,y$ 固定；点积 5 到 10；预测 10 到 15；损失 1 到 36”。它比“箭头更长、效果更差”更可复现。

若放大权重后写出预测 20，检查是否把偏置也乘了 2；若反转权重后写出 -10，检查固定偏置是否仍为 +5；若改变 $w_1$ 后点积仍为 5，检查程序是否真的替换索引 0。控制实验的解释力来自变量隔离。

反过来只改特征也能读权重：$x_1$ 从 2 到 3 时预测增加 4；恢复后令 $x_2$ 从 3 到 4，预测减少 1。两轮必须分开，因为同时修改后无法分别归因。这个观察只描述当前线性规则，不说明特征在所有数据上有因果效果。

还应记录图形表达的边界。贡献条只表达正负和数值，不宣称权重是几何方向；数值表负责表达单位与损失，颜色或动画不可成为唯一证据。

## 10. 误区诊断：看似合理的向量错误 {#vectors-misconceptions}

在阅读误区前做一次闭卷重建：先写坐标协议，再写两个 shape，然后展开点积，最后检查输出是否为标量。若只能记住箭头图，却无法说明 8 与 -3 从哪里来，说明几何直觉还没有接回预测；若只会算 5，却无法解释交换坐标的风险，说明运算尚未接回数据协议。

另一个自查是“最小反例”。要反驳“维度相同就一定有意义”，可举两个坐标单位完全不同的二维数组；要反驳“点积为零必有零向量”，可在独立几何例中举 `[1,0]` 与 `[0,1]`；要反驳“系数同比放大而预测同比放大”，可指出偏置没有缩放。能自己构造反例，比背定义更能揭示条件边界。

最后把单位写回贡献表。$w_1$ 若是分钟/基础任务，乘 $x_1$ 的基础任务数得到分钟；第二项与偏置也必须能相加为分钟。若两项单位无法相加，数值点积即使存在，模型定义也不完整。shape 检查负责数量，单位检查负责量纲，两者互补。

### 误区一：维度就是向量长度

**为什么容易发生：**日常语言把“长度”同时用于列表项数和几何距离。

**本例诊断：**独立几何位移 $u=[3,4]$ 的维度为 2，欧氏长度为 5，两者不是同一个数；主线特征单位不同，不把其范数解释为长度。

**修复动作：**写 `shape/维度` 与 `norm/长度` 两行账本，遇到“长度”先说明含义。

### 误区二：逐元素乘积就是点积

**为什么容易发生：**两者都包含配对乘法，而且 NumPy 都能运行。

**本例诊断：**`w*x` 得 `[8,-3]`，`w@x` 得 5；预期标量却得到向量就是诊断信号。

**修复动作：**先打印贡献，再显式 `.sum()`，并检查输出 shape。

### 误区三：把预测系数当作几何参考方向

**为什么容易发生：**点积也出现在几何投影公式中，容易忽略单位与尺度前提。

**本例诊断：**主线 $w_1,w_2$ 单位不同，职责是产生分钟贡献；独立几何例的 $u,v$ 才是无量纲同尺度位移。

**修复动作：**主线只写 $w_jx_j$、$w^Tx$ 与偏置；长度、夹角、投影只在明确满足几何条件的 $u,v$ 例中计算。

## 12. 小结与交接：从一个向量到一批样本 {#vectors-handoff}

本课建立了：一个样本是带顺序和语义的向量；主线 $w$ 是带单位线性泛函，只通过 $w_jx_j$ 与 $w^Tx$ 产生贡献；无量纲同尺度的独立 $u,v$ 例才使用长度、夹角和投影；目标只在预测后进入损失。

建议用四段时间完成本课：先用约 15 分钟重建坐标协议与点积，20 分钟逐步完成两套算例，15 分钟运行或手动跟踪代码与控制实验，最后用练习和误区做 20–30 分钟自查。若时间有限，不要跳过贡献表与 shape；独立几何投影可以稍后复看。学习证据不是读完页尾，而是能在空白纸上写出“语义顺序—配对贡献—标量点积—偏置预测—目标损失”的完整链。

交接前再验证三个最小案例：主线点积为 5；共同交换 $x,w$ 坐标后仍为 5；只交换 $x$ 后变为 10。它们分别证明计算正确、共同协议变更可保持结果、单边错位会破坏语义。若三个结果不能解释，矩阵堆叠只会把同一问题扩散到更多行。

把答案口述给同伴时，每个数都带上角色与单位，不说孤立的“等于 5”。这会暴露自己是否把点积、投影长度与最终分钟预测混在一起。

若没有同伴，就录下自己的两分钟解释，再对照贡献表检查是否漏说顺序、维度、偏置或目标边界；发现遗漏后重新解释一次。

矩阵课接收明确合同：`X = [[2, 3], [1, 4]]`，每一行是一个关卡，每一列沿用本课特征顺序。权重仍是 `w = [4, -1]`，偏置 `b = 5`。第一行应复现 `y_hat = 10`；第二行将得到 5。目标向量是 `[9, 7]`。

```text
输入：两个 shape (2,) 的样本行
组装：X shape (2, 2)
规则：每行分别与 w 做点积，再加同一个 b
待回答：怎样用一次 Xw+b 得到整批预测而不丢失 shape？
```

进入矩阵课前，请能从 `x,w,b` 重建 `y_hat`，再从 `y_hat,y` 重建 `L`；矩阵只是把同一规则稳定地扩展到多行。
