# 第 2 课：向量与样本表示——一次预测如何读懂多个特征

> **建议学习时间：60–90 分钟**
>
> **本课主线：**继续预测同一个小游戏关卡的预计完成时长。`x = [2, 3]`、`w = [4, -1]`、`b = 5`，预测 `y_hat = 10` 分钟，目标 `y = 9` 分钟。
>
> **学习方式：**每遇到紧凑符号都先展开为配对乘法；练习不计分，用于发现维度、顺序与方向上的理解缺口。

## 1. 开场：两个数为何能代表一个样本？ {#vectors-opening}

上一课把 `features = [2, 3]` 当作列表：2 个基础任务、3 次提示请求。现在请思考：为什么它可以写成一个整体 $\mathbf x$？若交换成 `[3, 2]`，还是同一个样本吗？为什么 `[4, -1] * [2, 3]` 不能直接叫预测？

真正的问题不是“会不会画箭头”，而是能否维护一份表示合同：每个位置有固定语义，维度一致，计算逐项配对，最终得到一个可解释输出。本课会用向量的顺序、维度、长度、方向和点积逐层回答。

先写猜想：$\mathbf w$ 更像“另一个样本”，还是“读取样本的规则”？当 $\mathbf w^T\mathbf x=5$ 时，这个 5 表示长度、方向，还是加权贡献？保留答案到实验节。

## 2. 回顾：有序列表、坐标与平方误差 {#vectors-recap}

向量课只需要三件旧知识。第一，列表有顺序：`x[0]` 始终对应基础任务数，`x[1]` 始终对应提示请求数。第二，坐标 $(2,3)$ 可画成从原点指向点 $(2,3)$ 的箭头；箭头只是表示法，不会改变数据。第三，本单元采用

$$
L=(\hat y-y)^2.
$$

平方使预测偏高与偏低都产生非负误差。主线样本中 $\hat y=10,y=9$，所以 $L=1$。这里先计算，不讨论怎样自动改变参数。

还要区分标量与向量：5、9、10 是标量；$[2,3]$ 和 $[4,-1]$ 是二维向量。标量只有一个位置，向量则是一组按协议排列的数。

## 3. 主线连接：向量把同一关卡的特征绑在一起 {#vectors-shared-task}

核心词汇始终是 `x`, `w`, `b`, `y_hat`, `y`, `L`：

```text
x = [2, 3]       当前关卡样本，shape (2,)
w = [4, -1]      模型权重，shape (2,)
b = 5            偏置，标量
y_hat = 10       预计完成时长，标量，单位分钟
y = 9            实际完成时长，标量，单位分钟
L = 1            当前样本平方误差，标量，单位分钟²
```

紧凑合同是 `y_hat = w^T x + b`。上标 $T$ 表示把 $\mathbf w$ 视为可与 $\mathbf x$ 配对的一行；在二维一维数组语境中，它等价于：

$$
\mathbf w^T\mathbf x=w_1x_1+w_2x_2.
$$

目标 $y$ 不进入点积。顺序固定为“先用 $x,w,b$ 得到 $y\_hat$，再用 $y\_hat,y$ 得到 $L$”。这条信息边界会贯穿后续课程。

## 4. 直觉：箭头、尺子与沿方向读取 {#vectors-intuition}

把 $\mathbf x=[2,3]$ 画成箭头时，横向 2、纵向 3 同时编码了方向和大小。长度为 $\sqrt{2^2+3^2}$，但机器学习中的“样本有多大”未必有直接物理含义：两个坐标单位不同，长度会受缩放影响。

$\mathbf w=[4,-1]$ 更像一把带方向的读取尺。第一个坐标按 4 倍计入，第二个按 -1 倍计入。点积为正，说明 $\mathbf x$ 在 $\mathbf w$ 指向的一侧有正的净投影；点积为零表示正交，两个方向在欧氏几何下没有沿彼此方向的分量。

投影只是辅助理解：

$$
\operatorname{proj}_{\mathbf w}(\mathbf x)
=\frac{\mathbf w^T\mathbf x}{\mathbf w^T\mathbf w}\mathbf w.
$$

预测主线真正需要的是点积这个标量，不必先画投影才能计算。尤其当特征经过不同尺度处理时，几何类比必须服从数据语义。

## 5. 正式表达：维度、长度、点积与投影 {#vectors-formal}

一个 $d$ 维实向量写为 $\mathbf x\in\mathbb R^d$。本例 $d=2$。维度是坐标个数，不是箭头长度：

$$
\dim(\mathbf x)=2,\qquad \lVert\mathbf x\rVert_2=\sqrt{x_1^2+x_2^2}=\sqrt{13}.
$$

点积要求维度相同：若 $\mathbf x,\mathbf w\in\mathbb R^d$，则

$$
\mathbf w^T\mathbf x=\sum_{j=1}^{d}w_jx_j\in\mathbb R.
$$

输入是两个长度为 $d$ 的向量，输出是标量。若 $\mathbf x=[2,3,1]$ 而 $\mathbf w=[4,-1]$，第三项没有权重，规则不完整。不能默默截断，也不能凭空补零。

夹角关系为 $\mathbf w^T\mathbf x=\lVert\mathbf w\rVert\lVert\mathbf x\rVert\cos\theta$。它解释方向，但主线中权重还携带“分钟/特征单位”，所以不要把权重只当无单位箭头。

### 深入一：顺序、单位与三种点积读法

向量坐标顺序属于数据接口，而不是排版选择。若第一列约定为基础任务、第二列为提示请求，那么把 `[2,3]` 改成 `[3,2]` 却继续使用 `[4,-1]`，得到的是语义错位的新记录。若确实交换两列，权重也必须按同一置换交换：

$$[4,-1]^T[2,3]=[-1,4]^T[3,2]=5,$$

但 $[4,-1]^T[3,2]=10$。点积对共同置换不变，对单边置换不变这一说法是错的。调试时应打印列名、特征值、权重与贡献四列，而不是只打印两个裸数组。

长度与角度也依赖尺度。若把“基础任务数”从个改成十个一组，2 变成 0.2，关卡事实没变，$\lVert x\rVert$ 却改变。任何几何结论都要附带坐标语义与尺度条件。权重能按相反比例补偿特征换算，所以参数值也必须连同特征单位解释。

同一点积有三种核验读法：配对贡献求和；$\lVert w\rVert\lVert x\rVert\cos\theta$ 的方向读取；行向量乘列向量 `(1,2) @ (2,1) -> (1,1)`。第一种最适合解释特征贡献，第二种提供几何直觉，第三种准备矩阵 shape。三者数值都应为 5，但返回容器可能是标量 5 或二维 `[[5]]`；实现必须规定一种接口。

### 深入二：代码 shape 与透明基线

`np.array([2,3])` 是 `(2,)`，`[[2,3]]` 是 `(1,2)`，`[[2],[3]]` 是 `(2,1)`。元素相同不代表运算合同相同。本课单样本统一采用 `(2,)`，所以点积直接返回标量。

```python
def dot_checked(w, x):
    if len(w) != len(x):
        raise ValueError("w and x must have the same dimension")
    total = 0.0
    for index, (weight, feature) in enumerate(zip(w, x)):
        contribution = weight * feature
        print(index, feature, weight, contribution)
        total += contribution
    return total
```

主线会打印 8、-3 并返回 5。这个慢但透明的版本是 NumPy `@` 的独立基线。输入还需检查有限值；`NaN` 或无穷值会穿过每个乘积，最后使预测与损失都不可解释。

## 6. 算例一：完整点积、预测与误差 {#vectors-worked-shared}

写下可核查纯文本：

```text
w^T x = 4*2 + (-1)*3 = 5
y_hat = 5 + 5 = 10
L = (10 - 9)^2 = 1
```

逐步看：$w_1x_1=4\times2=8$ 分钟，$w_2x_2=(-1)\times3=-3$ 分钟，净加权贡献为 $8-3=5$ 分钟；偏置 $b=5$ 分钟使预测为 10 分钟；目标 $y=9$ 分钟只在最后进入平方误差。

贡献表能定位符号错误：

| 坐标 | 特征 | 权重 | 乘积贡献 |
|---|---:|---:|---:|
| 1 | 2 | 4 | 8 |
| 2 | 3 | -1 | -3 |
| 汇总 | — | — | 5 |

核验输出形状：逐元素乘积 `[8, -3]` 是 `(2,)`；求和后 5 是标量；加偏置仍是标量。若代码停在 `[8, -3]`，计算尚未完成。

## 7. 算例二：长度与投影提供几何侧视图 {#vectors-worked-auxiliary}

继续用 $x=[2,3]$、$w=[4,-1]$，但本节只研究几何，不改主线变量。纯文本核验为 `||x|| = sqrt(13)`。同理 $\lVert\mathbf w\rVert=\sqrt{17}$，点积是 5，因此 $\mathbf x$ 在单位权重方向上的有符号长度为 `5/sqrt(17)`，约 1.213。

完整投影 `proj_w(x)` 为

$$
\frac{5}{17}[4,-1]=[20/17,-5/17].
$$

核验：投影与 $w$ 平行，因为它只是 $w$ 乘一个标量。剩余向量

$$
x-\operatorname{proj}_w(x)=[14/17,56/17]
$$

与 $w$ 点积为 $4(14/17)-56/17=0$。这说明点积确实只“读取”沿 $w$ 的分量。它不表示预测已经是 1.213；预测仍由未归一化权重的点积 5 再加偏置得到 10。

## 8. 代码翻译：保留 shape 与中间贡献 {#vectors-code}

```python
import numpy as np

x = np.array([2.0, 3.0])
w = np.array([4.0, -1.0])
b = 5.0
y = 9.0

assert x.shape == w.shape == (2,)
contributions = w * x
weighted_sum = w @ x
y_hat = weighted_sum + b
L = (y_hat - y) ** 2

print(contributions)  # [ 8. -3.]
print(weighted_sum)   # 5.0
print(y_hat, L)       # 10.0 1.0
```

`*` 做逐元素乘法，`@` 做点积。为了调试，可先保留两者；生产表达式可写 `y_hat = w @ x + b`。长度与投影：

```python
x_length = np.linalg.norm(x)
projection = (w @ x) / (w @ w) * w
```

若输入包含 `NaN` 或无穷值，结果会污染整条链。可先检查 `np.isfinite(x).all()`；若 shape 不同，明确报错，不用 `zip` 静默截断。

## 9. 控制实验：只旋转或缩放权重 {#vectors-experiment}

固定 $x=[2,3],b=5,y=9$，比较四个权重：

| $w$ | $w^Tx$ | $\hat y$ | $L$ | 观察 |
|---|---:|---:|---:|---|
| `[4,-1]` | 5 | 10 | 1 | 基线 |
| `[8,-2]` | 10 | 15 | 36 | 同方向放大两倍 |
| `[-4,1]` | -5 | 0 | 81 | 方向反转 |
| `[3,-1]` | 3 | 8 | 1 | 只改第一权重 |

这次一次只选择一种操作。整体缩放会保持方向而改变长度与点积；取负会反转方向；只改一项既改变方向也改变长度。预测变大不等于损失变小，评价必须回到目标 9。

建议画 $x$ 与四个 $w$，同时记录表格。图告诉你方向，表告诉你数值和单位；两种证据互相核对。不要从单个样本推断某个权重“全局最好”。

### 实验记录、变量隔离与形成性反馈

每轮实验应记录：改变因素、固定因素、点积、预测、目标、损失。以整体放大权重为例，应写“只把 $w$ 乘 2；$x,b,y$ 固定；点积 5 到 10；预测 10 到 15；损失 1 到 36”。它比“箭头更长、效果更差”更可复现。

若放大权重后写出预测 20，检查是否把偏置也乘了 2；若反转权重后写出 -10，检查固定偏置是否仍为 +5；若改变 $w_1$ 后点积仍为 5，检查程序是否真的替换索引 0。控制实验的解释力来自变量隔离。

反过来只改特征也能读权重：$x_1$ 从 2 到 3 时预测增加 4；恢复后令 $x_2$ 从 3 到 4，预测减少 1。两轮必须分开，因为同时修改后无法分别归因。这个观察只描述当前线性规则，不说明特征在所有数据上有因果效果。

还应记录图形类比的边界。整体缩放 $w$ 保持方向但点积改变，取负反转方向，只改一项同时改变方向与长度。图负责表达方向，数值表负责表达单位与损失；颜色或箭头动画不可成为唯一证据。

## 10. 误区诊断：看似合理的向量错误 {#vectors-misconceptions}

在阅读误区前做一次闭卷重建：先写坐标协议，再写两个 shape，然后展开点积，最后检查输出是否为标量。若只能记住箭头图，却无法说明 8 与 -3 从哪里来，说明几何直觉还没有接回预测；若只会算 5，却无法解释交换坐标的风险，说明运算尚未接回数据协议。

另一个自查是“最小反例”。要反驳“维度相同就一定有意义”，可举两个坐标单位完全不同的二维数组；要反驳“点积为零必有零向量”，可举 `[1,0]` 与 `[0,1]`；要反驳“方向相同预测就相同”，可比较 $w$ 与 $2w$。能自己构造反例，比背定义更能揭示条件边界。

最后把单位写回贡献表。$w_1$ 若是分钟/基础任务，乘 $x_1$ 的基础任务数得到分钟；第二项与偏置也必须能相加为分钟。若两项单位无法相加，数值点积即使存在，模型定义也不完整。shape 检查负责数量，单位检查负责量纲，两者互补。

### 误区一：维度就是向量长度

**为什么容易发生：**日常语言把“长度”同时用于列表项数和几何距离。

**本例诊断：**$x$ 的维度为 2，欧氏长度为 $\sqrt{13}$，两者不是同一个数。

**修复动作：**写 `shape/维度` 与 `norm/长度` 两行账本，遇到“长度”先说明含义。

### 误区二：逐元素乘积就是点积

**为什么容易发生：**两者都包含配对乘法，而且 NumPy 都能运行。

**本例诊断：**`w*x` 得 `[8,-3]`，`w@x` 得 5；预期标量却得到向量就是诊断信号。

**修复动作：**先打印贡献，再显式 `.sum()`，并检查输出 shape。

### 误区三：投影长度就是最终预测

**为什么容易发生：**“沿权重方向读取”听起来像模型的全部工作。

**本例诊断：**单位方向分量是 $5/\sqrt{17}$，原点积是 5，最终预测还要加 $b=5$ 得 10。

**修复动作：**分开写几何归一化量、模型点积与偏置后三列，不混用单位。

## 12. 小结与交接：从一个向量到一批样本 {#vectors-handoff}

本课建立了：一个样本是带顺序和语义的向量；点积完成配对乘法与汇总；长度、方向和投影帮助理解但不替代任务单位；目标只在预测后进入损失。

建议用四段时间完成本课：先用约 15 分钟重建坐标协议与点积，20 分钟逐步完成两套算例，15 分钟运行或手动跟踪代码与控制实验，最后用练习和误区做 20–30 分钟自查。若时间有限，不要跳过贡献表与 shape；几何投影可以稍后复看。学习证据不是读完页尾，而是能在空白纸上写出“语义顺序—配对贡献—标量点积—偏置预测—目标损失”的完整链。

交接前再验证三个最小案例：主线点积为 5；共同交换 $x,w$ 坐标后仍为 5；只交换 $x$ 后变为 10。它们分别证明计算正确、共同协议变更可保持结果、单边错位会破坏语义。若三个结果不能解释，矩阵堆叠只会把同一问题扩散到更多行。

把答案口述给同伴时，每个数都带上角色与单位，不说孤立的“等于 5”。这会暴露自己是否把点积、投影长度与最终分钟预测混在一起。

若没有同伴，就录下自己的两分钟解释，再对照贡献表检查是否漏说顺序、维度、偏置或目标边界；发现遗漏后重新解释一次。
