# UCI SMS 稀疏矩阵与 CSR matvec：中文字幕稿

视频无配音。以下内容按最终 75 秒成片的固定分镜记录教学叙事。

## 0:00–0:07

本节使用完整的 UCI SMS Spam Collection 本地快照，共 5,574 行。固定的文本处理规则得到一个 5,574×1,881 的 TF–IDF 矩阵，其中有 69,798 个非零项。本节不训练分类器，只解释文本怎样成为矩阵、CSR 怎样存储它，以及矩阵向量乘法怎样执行。

## 0:07–0:18

词表包含 1,881 个位置，但一条短信通常只出现其中很少的词。如果逐格存储，绝大部分位置都是 0。矩阵密度是 nnz 除以行数与列数的乘积，本例为 0.66571%；平均每行有 12.522 个非零项。稀疏首先是一种矩阵结构，不是模型效果的描述。

## 0:18–0:31

固定词表后，一条短信经过小写化和分词，变成固定宽度的一行。本例使用 TF–IDF 权重，词表只保留文档频率至少为 5 的词，每一行最后做 L2 归一化。画面中的短信只是教学示意，不是从数据集中摘录的个人消息。

## 0:31–0:46

CSR 用三条一维数组保存矩阵。data 存非零数值，indices 存这些数值所在的词表列，indptr 存每一行在前两条数组中的起止位置。画面按一位编号显示第 17 行，对应代码中的第 16 行。indptr 的相邻值是 283 和 299，因此真正访问的左闭右开切片是 [283,299)，共有 16 个非零项。

## 0:46–0:59

稠密写法把第 17 行的每一列与权重向量对应相乘。CSR 写法只遍历 k 从 283 到 298 的存储项，用 data[k] 乘以 w[indices[k]]。执行 Notebook 中的固定权重向量后，直接循环与 library CSR 都得到 −0.497805956，两者之差为 0。隐含的零没有被逐个读取，但线性代数结果没有改变。

## 0:59–1:09

如果使用稠密 float64，5,574×1,881 个位置共占 79.992 MiB，连 0 也占空间。实际 CSR 的 data、indices 和 indptr 共占 0.820 MiB，在本例中约小 97.55 倍。这个比例描述存储，不是运行速度承诺。

## 1:09–1:15

稀疏存储改变计算成本，不改变矩阵的数学意义。本地快照保留原始 5,574 行顺序，也保留 403 条重复消息。本节只完成表示、存储和 matvec；分类训练、去重策略与业务决策留给后续课程。
